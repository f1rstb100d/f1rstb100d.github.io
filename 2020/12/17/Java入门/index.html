Java入门.md

Posted on 2020-12-17 Edited on 2021-01-02

基础概念

  1. 跨平台原理：每种操作系统有自己相对应的JAVA虚拟机(JVM)来运行JAVA源码
  2. JRE(Java Runtime Environment)：Java运行时环境，包含JVM和运行所需的核心类库
  3. JDK(Java Development Kit)：Java开发工具包，包含JRE和开发工具(编译工具javac.exe和运行工具java.exe)
  4. IDEA快捷键：psvm快速生成main函数，sout快速生成输出语句，Ctrl+Alt+Space内容提示代码补全，Ctrl+/单行注释，Ctrl+Shift+/多行注释，Ctrl+Alt+L格式化
  5. System.out.println();会自动换行，System.out.print();是不会换行
  6. 看方法的源码，选中方法，按下Ctrl+B
  7. 有红色下划线，Alt+Enter查看推荐修改方式
  8. Alt+7可以打开一个窗口来预览该类下所有方法名的索引
  9. 输入等号右边需要调用的函数，Ctrl+Alt+V可以根据返回值自动生成左边的赋值变量类型和变量名

关键字

关键字全部都是小写字母

常量

程序运行过程中，值不可以发生改变的量

分类：字符串常量（用双引号括起来的内容）、整数常量（不带小数的数字）、小数常量（带小数的数字）、字符常量（用单引号括起来的内容）、布尔常量（只有true和false）、空常量（null）

空常量是不能直接print的，其他的都可以

数据类型

存储设备最小信息单元叫“位（bit）”，由“b”表示；计算机最小存储单元叫“字节（byte）”，由“B”表示，字节是8个位组成。1B=8bit，1KB=1024B，1MB=1024KB

Java是强类型语言，每一种数据有明确的数据类型，不同数据类型对应不同内存空间

数据类型 关键字 内存占用
整数 byte 1
整数 short 2
整数 int(默认) 4
整数 long 8
浮点数 float 4
浮点数 double(默认) 8
字符 char 2
布尔 boolean 1

变量

程序运行过程中，值可以改变的量

定义：int a = 10;

取值：a

修改值：a = 20;

注意long l = 10000L; 以及float f = 13.14F;

标识符

给类、方法、变量起名字的符号

规则：由数字、字母、下划线和$符号组成，不能以数字开头，不能是关键字，区分大小写

约定：

小驼峰命名法(针对方法和变量)：标识符是一个单词最好首字母小写(name)，标识符多个单词组成最好第一个单词首字母小写其他单词首字母大写(firstName)

大驼峰命名法(针对类名)：标识符是一个单词最好首字母大写(Student)，标识符多个单词组成最好每个单词首字母都大写(GoodStudent)

类型转换

自动类型转换：把表示范围小的变量赋值给另一个表示范围大的变量

从小到大：byte->short->int->long->float->double以及char->int->long->float->double

强制类型转化：把表示范围大的变量赋值给一个表示范围小的变量。如：int k = (int)88.88;

算术运算符

a + b, a - b, a * b, a / b（得到商）,a % b（得到余数）

整数相除只能得到整数，要想得到浮点数至少有一个浮点数参与运算

字符的+操作：10 + 'A' = 75，因为’A’的ASCII码是65，’a’的ASCII是97，’0’的ASCII码是48。整个表达式的结果自动提升到表达式中最高等级操作数同样类型即char->int

字符串的+操作："F1rst" + "b100d" = "F1rstb100d"以及"F1rst" + 123(int) = "F1rst123"

逻辑运算符

&, |, ^, !：逻辑与或异或非，例如(a>b)&(a>c)

^ 逻辑异或 a^b，相同取0，不同取1

符号 作用 说明

方法重载

方法重载指同一个类中定义的多个方法之间的关系，满足下列条件的多个方法构成重载

  1. 多个方法在同一个类中
  2. 多个方法具有相同的方法名
  3. 多个方法的参数不同，类型不同或者数量不同

Java虚拟机会通过参数的不同来区分同名的方法

方法的参数传递

对于基本类型数据的参数，形式参数的改变，不影响实际参数的值

对于引用类型数据，形参的修改会影响实参

继承

继承是面向对象三大特征之一。可以使得子类具有父类的属性和方法，还可以在子类中重新定义，追加属性和方法

super关键字的用法和this关键字的用法相似：

this：代表本类对象的引用

super：代表父类存储空间的标识

继承中构造方法的特点：

子类中所有的构造方法默认都会访问父类中无参的构造方法。因为子类会继承父类的数据，所以子类初始化之前一定要先完成父类的初始化，子类构造方法第一句默认都是super()

方法重写：子类中出现和父类一模一样的方法声明

方法重写注意事项：

父类中私有方法private是不能被重写的

子类重写之后的方法权限不能更低（public>默认什么都不加>private）

Java中继承的注意事项：

Java只支持单继承，不支持多继承，即extends只能接一个父类

Java中类支持多层继承，Son extends Father, Father extends Granddad

状态修饰符（靠后）

final（最终态）：可以修饰成员方法，成员变量，类。final修饰的成员方法继承之后不能再被重写了，final修饰的成员变量不能再被赋值了（成常量了），final修饰的类不能再被继承了。

static（静态）：可以修饰成员方法，成员变量。static修饰的内容被类的所有对象共享，可以通过最开始类名.成员变量赋值，后面所有new的对象就不用特意去传这个参数了。静态修饰意味着可以通过类名直接访问

静态方法只能访问静态成员

final修饰局部变量：

变量是基本类型，final指基本类型的数据值不能改变。

变量是引用类型，final指引用类型的地址值不能改变，但引用里面的内容还是可以变的。

多态

同一个对象，在不同时刻表现出来的不同形态

前提：有继承实现关系，有方法重写，有父类引用指向子类对象

多态中成员访问特点

成员变量：编译看左边，执行看左边

成员方法：编译看左边Animal，执行看右边Cat

因为成员方法有重写，所以要看右边

也就是说，可以写个接收Animal类型的方法做为通法，传什么动物进来编译都不会报错，执行看右边即调用eat方法是执行动物各自的eat方法

多态中的转型

向上转型：从子到父，父类引用指向子类对象Animal a = new Cat();

向下转型：从父到子，父类引用转为子类对象Cat c = (Cat)a;

抽象类

一个没有方法体的方法应该被定义为抽象方法，而类中如果有抽象方法，该类必须定义为抽象类

接口

接口成员变量：只能是常量，默认修饰符public static final int a = 1;

接口构造方法：接口没有构造方法，如果没有父类则默认继承自Object类

接口成员方法：只能是抽象方法，默认修饰符public abstract void jump();

类的extends只能接一个类来继承

但接口的extends可以接多个接口来继承

并且implements inter1,inter2,inter3 也可以同时实现多个接口

抽象类和接口的区别

成员区别

抽象类 常量,变量,构造方法,抽象方法,非抽象方法
接口 常量,抽象方法

关系区别

类与类 继承,单继承
类与接口 实现,单实现,多实现
接口与接口 继承,单继承,多继承

设计理念

抽象类 对类抽象，包括属性和行为
接口 对行为抽象，主要是行为

类加载器

类的加载：

将class文件读入内存，并创建一个java.lang.Class对象

类的连接：

验证阶段：验证加载的类是否具有正确的内部结构，并和其他类协调一致

准备阶段：负责为类的类变量分配内存，并设置默认初始化值

解析阶段：将类的二进制数据中的符号引用替换为直接引用

类的初始化：

主要就是对类的变量进行初始化

步骤：加入类还未被加载个连接，则程序先加载并连接该类；加入该类的直接父类还未被初始化，则先初始化其直接父类；加入类中有初始化语句，则系统以此执行这些初始化语句

类的初始化时机：创建类的实例时；调用类的类方法；访问类或者接口的类变量，或者为该类变量赋值；使用反射方法强制创建某个类或接口对应的java.lang.Class对象时；初始化某个类的子类时；直接使用java.exe命令来运行某个主类

JVM的类加载机制：

全盘负责：该Class所依赖和引用的其他Class也由该类加载器负责载入

父类委托：先让父类加载器试图加载该Class，只有父类加载器失败时才尝试从自己路径加载该类

缓存机制：所有加载过的Class都会被缓存，先从缓存区搜索该Class，当找不到时系统会读取该类的二进制数据并转换成Class对象，并保存到缓存区

ClassLoader：是负责加载类的对象

Java运行时具有以下内置类加载器：Bootstrap class loader（爷），Platform class loader（父），System class loader（子）

ClassLoader中的两个方法：

static ClassLoader getSystemClassLoader()：返回用于委派的系统类加载器

ClassLoader getParent()：返回父类加载器进行委派
